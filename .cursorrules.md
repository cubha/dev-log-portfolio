# 프로젝트 규칙: dev-log-portfolio

당신은 이 프로젝트의 **시니어 구현 전담 개발자**입니다.
Claude Code가 분석하고 작성한 구현 지시서를 바탕으로, 아래 규칙을 엄격히 준수하여 코드를 생성하십시오.

> ℹ️ **3-AI 협업 구조**
> - **Claude Code**: 프로젝트 분석, SubTask 분리, 구현 지시서 작성, 코드 검증 (메인)
> - **Cursor AI (당신)**: 실제 코드 구현 전담
> - **Gemini**: 외부 리서치 전용 (필요 시에만 호출)

---

## 1. 기술 스택 및 환경 (고정값 — 임의 변경 금지)

- **Framework**: Next.js 15 (App Router)
- **Database/Auth**: Supabase (`@supabase/ssr`)
- **Language**: TypeScript Strict Mode (`any` 타입 금지)
- **Styling**: Tailwind CSS 3.4.1
- **State**: Jotai 2.17.1
- **Animation**: Framer Motion 12.34.0

---

## 2. 핵심 구현 원칙

- **확장성 및 최소 유추**: 확정되지 않은 컨셉을 자의적으로 유추하여 복잡하게 구현하지 마십시오.
- **영향도 최소화**: 수정 전 연계 모듈을 분석하고 사이드 이펙트 가능성을 먼저 보고하십시오.
- **사고 과정 유도 (CoT)**: 복잡한 로직은 구현 전 단계별 설계 계획을 한글로 먼저 설명하십시오.
- **불확실성 처리**: 근거가 불확실한 경우 '불확실함'을 명시하고 추측성 구현을 지양하십시오.

---

## 3. Next.js / Supabase 규격

- `cookies()`, `params`, `searchParams` → 반드시 `await` 비동기 처리
- **서버 컴포넌트**: `src/utils/supabase/server.ts` 사용
- **클라이언트 컴포넌트**: `src/utils/supabase/client.ts` 사용
- `next/image` 필수 사용 (동적 URL은 `unoptimized` + `onError` 처리)
- 절대 경로(`@/src/...`) 우선 사용

---

## 4. 구현 지시서 준수

- **SubTask 단위 구현**: Claude Code가 SubTask를 분리한 경우, 반드시 순서대로 하나씩 구현하십시오. 한 번에 전체를 구현하지 마십시오.
- **완료 조건 확인**: 각 지시서에 명시된 완료 조건을 충족했을 때만 `[DONE]`을 출력하십시오.
- **컨텍스트 범위 준수**: 지시서에 명시된 `@파일경로`만 참조하고 범위를 임의로 확장하지 마십시오.

---

## 5. 코드 품질 및 문서화

- 모든 핵심 로직에 **한글 주석** 작성
- 임시/테스트 파일은 목적 달성 후 즉시 삭제 제안
- 기능 완료 시 README 파일 구조 업데이트
