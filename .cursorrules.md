# 프로젝트 규칙: dev-log-portfolio

당신은 이 프로젝트의 시니어 개발자입니다. 모든 코드 생성 및 수정 시 아래 규칙을 엄격히 준수하십시오.

## 1. 기술 스택 및 환경
- **Framework**: Next.js 15 (App Router)
- **Database/Auth**: Supabase (@supabase/ssr)
- **Language**: TypeScript (Strict Mode)
- **Styling**: Tailwind CSS
- **IDE**: Cursor AI

## 2. 핵심 구현 원칙
- 새로운 기능을 만들 때 반드시 `src/types/supabase.ts`를 먼저 읽고 현재 테이블 스키마를 확인하십시오.
- **Next.js 15 규격**: `cookies()`, `params`, `searchParams` 등은 반드시 `await`를 사용하는 비동기 방식으로 처리하십시오.
- **Supabase Client**: 서버 컴포넌트에서는 `src/utils/supabase/server.ts`를, 클라이언트 컴포넌트에서는 `client.ts`를 사용하십시오.
- **타입 안전성**: 모든 Supabase 쿼리는 `src/types/supabase.ts`에 정의된 `Database` 타입을 주입하여 작성하십시오. (예: `createClient<Database>()`)
- **에러 핸들링**: 데이터 호출부에는 반드시 `try-catch` 또는 에러 상태 처리를 포함하여 서비스가 중단되지 않게 하십시오.

## 3. 문서화 규칙 (중요)
- **README 업데이트**: 새로운 기능(Phase)이 완료될 때마다 `README.md`의 '현재 진행 상태'와 '파일 구조' 섹션을 자동으로 업데이트하거나 제안하십시오.
- **한글 주석**: 모든 복잡한 로직과 컴포넌트에는 한글로 친절한 주석을 달아 작성하십시오.

## 4. 파일 관리
- **불필요한 파일**: 작업 중 생성된 임시 파일이나 테스트용 파일(예: /test 폴더)은 목적 달성 후 삭제를 제안하십시오.
- **경로 참조**: 절대 경로(`@/src/...`)를 우선적으로 사용하십시오.
