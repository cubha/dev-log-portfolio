# 🚀 개인 포트폴리오 (Next.js 15 & Supabase)

> **단순 소개 페이지를 넘어선 풀스택 포트폴리오**
> 실시간 GitHub 활동, Spotify Now Playing, 방명록, 관리자 대시보드까지 완벽하게 통합한 프로젝트입니다.

---

## ✨ 핵심 구현 내용

### 1. 🐙 GitHub Live Status (GraphQL API)
REST API의 한계를 극복하고, Private 레포지토리 커밋까지 포함한 **정확한 기여도 집계**를 구현했습니다.
- `repo` + `read:user` 스코프 PAT 발급
- `contributionsCollection.totalCommitContributions` 필드로 연간 커밋 수 조회
- 서버사이드 전용 호출로 클라이언트 토큰 노출 완벽 차단

### 2. 🔐 Supabase 인증 및 관리자 시스템
- **JWT 기반 인증:** Supabase Auth 이메일/패스워드 로그인 및 세션 발급
- **권한 분리:** `profiles` 테이블 `role` 컬럼으로 관리자 권한 제어
- **보안 라우팅:** Next.js 15 Middleware에서 JWT 검증 및 role 기반 라우트 가드 적용
- **소셜 연동:** GitHub OAuth를 활용한 방명록 소셜 로그인 지원

### 3. 🛠 프로젝트 CRUD 관리자 대시보드
- 관리자 전용 프로젝트 등록/수정/삭제 기능
- **Storage 관리:** Supabase Storage 이미지 업로드 (드래그 앤 드롭 지원, 5MB 제한)
- **동적 데이터 처리:** `detailed_tasks` 리스트 입력 (`|||` 구분자 배열 처리)
- **UI 최적화:** `project_type` 기반 'LIVE DEMO' 버튼 조건부 노출

### 4. 📝 방명록 (Guestbook)
- 익명 / GitHub OAuth / 관리자 3가지 인증 모드 지원
- **데이터 보안:** Supabase RLS(Row Level Security) 정책으로 삭제 권한 엄격 제어
- **UX 개선:** 페이지네이션 (10개/페이지) 및 Server Action 기반 CRUD 구현

### 5. 🎨 프로젝트 리스트 UI 고도화
- **반응형 레이아웃:** X축 드래그 슬라이더에서 Y축 반응형 CSS Grid로 전환 (`grid-cols-1 md:grid-cols-2 lg:grid-cols-3`)
- **인터랙션:** Framer Motion `whileInView` Fade-up 애니메이션 적용 (스크롤 진입 시 순차 노출)
- **상태 관리:** Jotai `projectFilterAtom`을 활용한 카테고리 필터링
- **라우팅:** 프로젝트 모달에서 MDX 기반 상세 페이지로 부드러운 전환

## 🚨 트러블슈팅

### 1️⃣ GitHub Private 레포 커밋 집계 불가
* **문제 상황:** REST Events API(`/users/{username}/events`)는 Private 레포 `PushEvent`에서 `payload.size`, `payload.commits` 필드를 모두 제거함
* **원인 분석:** REST API의 보안 및 프라이버시 정책으로 인한 페이로드 누락
* **해결 방안:** **GitHub GraphQL API**의 `viewer.contributionsCollection`으로 대체하여 Private 포함 정확한 연간 커밋 수 확보

### 2️⃣ Supabase RLS SELECT 정책 무력화
* **문제 상황:** `USING(true)` 정책이 하나라도 존재하면 OR 연산으로 인해 다른 제한 정책이 무효화됨
* **원인 분석:** Supabase(PostgreSQL) RLS의 정책 평가 방식이 다중 정책을 OR 조건으로 합산하기 때문
* **해결 방안:** 공개/비공개 **분리 정책 조합 방식**으로 재설계하여 단일 `USING(true)` 남용 방지 및 권한 제어 강화

### 3️⃣ Framer Motion 신규 카드 미작동
* **문제 상황:** 이미 마운트된 부모 내에 새로 추가된 자식 컴포넌트는 부모의 variant 전파를 받지 못함
* **원인 분석:** React 생명주기와 Framer Motion의 DOM 진입 감지 시점 불일치
* **해결 방안:** 각 카드 컴포넌트에 `initial` / `whileInView`를 **명시적으로 직접 선언**하여 독립적으로 동작하도록 수정
